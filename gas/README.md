# Инструкция по настройке Google Apps Script (Бэкенд)

Этот бэкенд использует бесплатную платформу Google Apps Script для развертывания веб-приложения, а в качестве базы данных выступает Google Таблица.

---

## Шаг 1: Создание Google Таблицы

1. Откройте [Google Таблицы](https://sheets.google.com) и создайте пустую таблицу.
2. Придумайте название (например, `Suluu Business DB`).
3. Скопируйте **ID таблицы** из адресной строки браузера.
   * *Пример:* в ссылке `https://docs.google.com/spreadsheets/d/1A2B3C4D.../edit` ID таблицы — это `1A2B3C4D...`.
   * *Примечание:* По умолчанию скрипт привязан к текущей таблице, так что указывать ID вручную не требуется.

---

## Шаг 2: Открытие редактора скриптов

1. В меню вашей Google Таблицы выберите **Расширения → Apps Script** (Extensions → Apps Script).
2. Перед вами откроется встроенный редактор кода. Удалите весь существующий код в файле `Код.gs` (по умолчанию пустая функция `myFunction`).

---

## Шаг 3: Перенос кода бэкенда

Создайте в редакторе Apps Script следующие файлы (нажимая кнопку **+** рядом с разделом "Файлы" и выбирая тип "Скрипт"):

1. **`Code`** (или переименуйте `Код.gs` в `Code.gs`)
   * Скопируйте содержимое файла `business/gas/Code.gs` с вашего компьютера в этот файл.
2. **`Auth`** (создайте `Auth.gs`)
   * Скопируйте содержимое файла `business/gas/Auth.gs` в этот файл.
3. **`Database`** (создайте `Database.gs`)
   * Скопируйте содержимое файла `business/gas/Database.gs` в этот файл.
4. **`Bookings`** (создайте `Bookings.gs`)
   * Скопируйте содержимое файла `business/gas/Bookings.gs` в этот файл.
5. **`Masters`** (создайте `Masters.gs`)
   * Скопируйте содержимое файла `business/gas/Masters.gs` в этот файл.
6. **`Clients`** (создайте `Clients.gs`)
   * Скопируйте содержимое файла `business/gas/Clients.gs` в этот файл.
7. **`Services`** (создайте `Services.gs`)
   * Скопируйте содержимое файла `business/gas/Services.gs` в этот файл.
8. **`Finance`** (создайте `Finance.gs`)
   * Скопируйте содержимое файла `business/gas/Finance.gs` в этот файл.
9. **`Settings`** (создайте `Settings.gs`)
   * Скопируйте содержимое файла `business/gas/Settings.gs` в этот файл.

*Нажмите значок **Сохранить** (дискету) в верхней панели после создания всех файлов.*

---

## Шаг 4: Деплой веб-приложения

1. Нажмите синюю кнопку **Начать развертывание → Новое развертывание** (Deploy → New deployment) в правом верхнем углу.
2. Нажмите на значок шестеренки рядом с типом развертывания и выберите **Веб-приложение** (Web app).
3. Заполните настройки:
   * **Описание:** `Suluu Business v1`
   * **Запуск от имени:** **Я (ваша почта)** (Execute as: Me)
   * **Кто имеет доступ:** **Все** (Anyone) — *это критически важно, чтобы фронтенд мог отправлять запросы.*
4. Нажмите кнопку **Развернуть** (Deploy).
5. При первом развертывании Google попросит вас предоставить доступ к аккаунту:
   * Нажмите **Предоставить доступ** (Authorize access).
   * Выберите свой аккаунт Google.
   * Появится окно предупреждения о безопасности ("Приложение не проверено"). Нажмите снизу кнопку **Дополнительно** (Advanced), а затем — **Перейти к проекту (небезопасно)** (Go to ... (unsafe)).
   * Нажмите **Разрешить** (Allow).
6. После успешного развертывания скопируйте **URL веб-приложения** (Web app URL).
   * Он будет выглядеть так: `https://script.google.com/macros/s/AKfycb.../exec`.

---

## Шаг 5: Первый запуск и пароль

1. Откройте фронтенд приложения (развернутый на GitHub Pages или запущенный локально).
2. Вставьте скопированный **URL веб-приложения** в поле настройки.
3. Придумайте пароль администратора. При первом входе введенный вами пароль автоматически захешируется и сохранится в настройках скрипта в качестве постоянного пароля администратора.
4. Вход завершен! Вы можете управлять вашим бизнесом.

---

## Устранение неполадок (CORS / Ошибки авторизации)

1. **Ошибка CORS при подключении:** Убедитесь, что в Apps Script при деплое в поле "Кто имеет доступ" выбрано именно "Все" (Anyone), а не "Только я". Если вы изменили настройки, вам обязательно нужно создать **Новую версию развертывания** (Manage deployments → Edit → New version → Deploy).
2. **Как сбросить пароль администратора:**
   * В Apps Script перейдите в **Настройки проекта** (шестеренка в левой панели).
   * В самом низу в разделе "Свойства сценария" (Script Properties) удалите свойство `ADMIN_PASSWORD_HASH`.
   * Обновите фронтенд и при следующем входе вы сможете задать пароль заново.
